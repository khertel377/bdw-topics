HTML Reflow

Reflow is the name of the process by which a browser recalculates the positions and sizes of elements on the page in order to re-render part or all of the document. The HTML formatting objects are called frames, and the browser will place these frames hierarchically based on the content model provided by the document. A reflow is triggered when the input information used to construct a render tree is changed.

Some actions that trigger a reflow:

  • Resizing the window
  • Changing the font
  • Adding or removing stylesheet
  • Content changes, such as a user typing text in
  • An input box
  • Activation of CSS pseudo classes such as :hover
  • Manipulating the class attribute
  • A script manipulating the DOM
  • Calculating offsetWidth and offsetHeight

Reflows greatly affect speed and performance, which ultimately harm the user experience. To minimize these effects, developers have some basic guidelines.

  1. Reduce unnecessary DOM depth. Changes at one level in the DOM tree can cause changes at every level of the tree - all the way up to the root, and all the the way down into the children of the modified node. This leads to more time being spent performing reflow.
  2. Minimize CSS rules, and remove unused CSS rules.
  3. If you make complex rendering changes such as animations, do so out of the flow. Use position-absolute or position-fixed to accomplish this.
  4. Avoid unnecessary complex CSS selectors - descendant selectors in particular - which require more CPU power to do selector matching.
